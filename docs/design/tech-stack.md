## 3. 前端技术选型

## 3.1 为什么选择 Vue 3

选择 `Vue 3` 的原因：

- 组件化开发清晰，适合中小型项目快速搭建
- 生态成熟，和 `Vue Router`、`Pinia`、`Axios` 配合自然
- 适合同时支撑移动端 H5 和 Web 管理端
- 学习成本和开发复杂度低于更重的方案

对本项目的适配性：

- 可以统一实现移动端 H5 用户界面和 Web 后台式界面
- 方便做角色区分和路由权限控制
- 有利于控制毕设开发周期

## 3.2 为什么选择 Vite

选择 `Vite` 的原因：

- 启动速度快
- 开发体验轻量
- 配置简单，适合毕设项目

## 3.3 为什么选择 Vue Router

选择 `Vue Router` 的原因：

- 负责前端页面路由管理
- 便于实现登录拦截、角色路由和页面跳转
- 适合移动端与 Web 端共用路由组织思路

## 3.4 为什么选择 Pinia

选择 `Pinia` 的原因：

- 用于管理登录态、用户信息、购物车状态等全局数据
- 相比更旧的状态管理方案更简洁
- 更适合 `Vue 3`

## 3.5 为什么选择 Axios

选择 `Axios` 的原因：

- 用于调用后端 REST API
- 使用简单，适合统一封装请求、响应和 token 拦截

## 3.6 为什么 Web 端选择 Element Plus

选择 `Element Plus` 的原因：

- 适合后台管理类系统
- 表格、表单、弹窗、分页等组件齐全
- 能快速搭建商品管理、订单管理、库存管理、统计页面

## 3.7 为什么移动端采用 Vue H5

选择移动端 H5 的原因：

- 不需要单独接入小程序
- 与 Web 端共用一套前端技术栈
- 适合做顾客购物页面
- 开发量明显小于原生 App 或多端并行方案

本项目前端最终采用：

- 一套 `Vue 3` 技术体系
- 两套 UI 形态
  - 移动端 H5 UI
  - Web 后台式 UI

## 4. 后端技术选型

## 4.1 为什么选择 Django

选择 `Django` 的原因：

- 框架成熟稳定
- 自带 ORM、后台管理、用户系统等基础能力
- 适合快速构建中小型业务系统
- 对本科毕设来说，开发效率高，结构清晰

## 4.2 为什么选择 Django REST Framework

选择 `Django REST Framework` 的原因：

- 适合构建 REST API
- 序列化、权限、分页、视图集能力完整
- 便于前后端分离开发
- 能较好支撑用户端和商家端接口统一管理

## 4.3 为什么选择 JWT

选择 `JWT` 的原因：

- 适合前后端分离项目
- 方便前端存储登录态并在请求头中携带
- 便于实现用户端和商家端的角色权限区分

## 5. 数据库技术选型

## 5.1 为什么选择 MySQL

选择 `MySQL` 的原因：

- 关系型数据库成熟稳定
- 与 `Django ORM` 配合良好
- 适合订单、库存、商品、用户等结构化业务数据
- 资料丰富，适合毕业设计使用

对本项目的适配性：

- 支持订单和库存这类强结构业务数据
- 适合做多表关联和统计查询

## 6. 图像识别技术选型

## 6.1 为什么选择 YOLO

选择 `YOLO` 的原因：

- 适合图像检测与识别场景
- 在“商品图片辅助识别”这个亮点功能上具备展示价值
- 易于作为单独识别模块接入后端服务

## 6.2 在本项目中的使用边界

`YOLO` 在本项目中只用于：

- 商家上传商品图片后的辅助识别
- 输出商品名称候选、推荐分类和置信度

`YOLO` 不负责：

- 自动保存商品
- 自动上架商品
- 替代商家人工确认

这样设计的原因：

- 控制项目复杂度
- 降低识别误差对业务的影响
- 更适合作为毕设中的智能化增强功能

## 7. 数据可视化选型

## 7.1 为什么选择 ECharts

选择 `ECharts` 的原因：

- 适合实现销售统计和热销商品排行
- 使用广泛，图表展示效果好
- 适合答辩展示

## 8. 技术选型的总体取舍

本项目技术选型遵循以下原则：

- **优先可实现**
  - 不引入不必要的复杂技术
- **优先完整性**
  - 保证用户端、商家端、库存、订单、识别功能都能闭环
- **优先展示效果**
  - 在保证可实现的前提下体现毕设亮点
- **优先统一技术栈**
  - 前端统一使用 `Vue 3`
  - 后端统一使用 `Django`

## 9. 选型结论

本项目最终采用：

- 前端：`Vue 3` + `Vite` + `Vue Router` + `Pinia` + `Axios`
- Web UI：`Element Plus`
- 移动端：`Vue 3 H5`
- 后端：`Django` + `Django REST Framework`
- 认证：`JWT`
- 数据库：`MySQL`
- 图像识别：`YOLO`
- 图表：`ECharts`

该方案能够满足：

- 本科毕设的开发复杂度要求
- 前后端分离开发模式
- 用户端与商家端双系统设计
- 图像识别辅助录入这一功能亮点
