# 用户端 API 文档

本文档整理前端用户端会调用的接口，重点补充请求字段与返回字段。以下内容以当前后端实际代码为准。

说明：
- 接口基础前缀为 `/api`
- 需要登录的接口应在请求头中携带认证信息
- 未特别说明时，返回体为 `application/json`

## 1. 用户认证

### 1.1 用户注册
- **URL**: `POST /api/auth/register`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `username` | 是 | 用户名，用于登录和展示 |
| `password` | 是 | 登录密码 |
| `phone` | 否 | 手机号 |
| `email` | 否 | 邮箱地址 |
| `role` | 否 | 用户角色；用户端通常不传，默认 `user` |

**返回字段**
- `201 Created`
- 返回 `User` 对象，字段见“6.1 用户对象”

### 1.2 用户名密码登录
- **URL**: `POST /api/auth/login/password`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `username` | 是 | 用户名 |
| `password` | 是 | 登录密码 |

**返回字段**
- `200 OK`

| 字段名 | 含义 |
|---|---|
| `token` | 当前登录态 token，格式为 `mock-token-{user.id}` |
| `user` | 用户对象，字段见“6.1 用户对象” |

### 1.3 手机验证码登录
- **URL**: `POST /api/auth/login/code`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `phone` | 是 | 用户手机号 |
| `code` | 是 | 手机验证码，当前后端仅接受 `123456` |

**返回字段**
- `200 OK`

| 字段名 | 含义 |
|---|---|
| `token` | 当前登录态 token，格式为 `mock-token-{user.id}` |
| `user` | 用户对象，字段见“6.1 用户对象” |

### 1.4 获取当前用户信息
- **URL**: `GET /api/auth/profile`

**请求字段**
- 无

**返回字段**
- `200 OK`
- 返回 `User` 对象，字段见“6.1 用户对象”

### 1.5 修改当前用户信息
- **URL**: `PUT /api/auth/profile`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `username` | 否 | 新用户名 |
| `phone` | 否 | 新手机号 |
| `email` | 否 | 新邮箱 |
| `avatar` | 否 | 用户头像地址或头像标识 |

**返回字段**
- `200 OK`
- 返回更新后的 `User` 对象，字段见“6.1 用户对象”

### 1.6 上传头像
- **URL**: `POST /api/auth/avatar`

**请求类型**
- `multipart/form-data`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `avatar` | 是 | 用户上传的头像文件 |

**返回字段**
- `200 OK`
- 返回更新后的 `User` 对象，字段见“6.1 用户对象”

### 1.7 修改密码
- **URL**: `PUT /api/auth/password`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `old_password` | 是 | 当前旧密码 |
| `new_password` | 是 | 新密码 |

**返回字段**
- `204 No Content`
- 返回体为空

## 2. 首页与商品

### 2.1 获取首页内容
- **URL**: `GET /api/home`

**请求字段**
- 无

**返回字段**
- `200 OK`

| 字段名 | 含义 |
|---|---|
| `announcement` | 当前发布中的公告对象；如果没有已发布公告则为 `null` |
| `recommendations` | 推荐商品列表，元素字段见“6.5 推荐商品对象” |

### 2.2 获取商品分类列表
- **URL**: `GET /api/categories`

**请求字段**
- 无

**返回字段**
- `200 OK`
- 返回数组，数组元素字段见“6.2 分类对象”

### 2.3 获取商品列表
- **URL**: `GET /api/products`

**查询字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `category_id` | 否 | 按分类 ID 筛选商品 |
| `keyword` | 否 | 按商品名称或描述关键字搜索 |
| `page` | 否 | 分页页码，从 `1` 开始；不传时返回全部 |
| `size` | 否 | 每页数量，默认 `10`，最大 `50` |

**返回字段**
- `200 OK`
- 返回数组，数组元素字段见“6.3 商品对象”

**分页说明**
- 当前接口采用“查询参数分页 + 数组返回”的方式，返回体不包含 `total` 等分页元信息。
- 前端可通过“本次返回数量 < size”判断是否还有下一页。

### 2.4 获取商品详情
- **URL**: `GET /api/products/{product_id}`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `product_id` | 是 | 商品 ID |

**返回字段**
- `200 OK`
- 返回 `Product` 对象，字段见“6.3 商品对象”

## 3. 购物车

### 3.1 获取当前用户购物车
- **URL**: `GET /api/cart`

**请求字段**
- 无

**返回字段**
- `200 OK`
- 返回 `Cart` 对象，字段见“6.6 购物车对象”

### 3.2 添加商品到购物车
- **URL**: `POST /api/cart/items`

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `product_id` | 是 | 要加入购物车的商品 ID |
| `quantity` | 是 | 加入数量，最小为 1 |

**返回字段**
- `200 OK`
- 返回 `CartItem` 对象，字段见“6.7 购物车项对象”

### 3.3 修改购物车商品
- **URL**: `PUT /api/cart/items/{item_id}`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `item_id` | 是 | 购物车项 ID |

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `quantity` | 是 | 商品数量，最小为 1 |
| `selected` | 否 | 是否选中用于结算 |

**返回字段**
- `200 OK`
- 返回更新后的 `CartItem` 对象，字段见“6.7 购物车项对象”

### 3.4 删除购物车商品
- **URL**: `DELETE /api/cart/items/{item_id}`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `item_id` | 是 | 购物车项 ID |

**返回字段**
- `204 No Content`
- 返回体为空

### 3.5 获取结算信息
- **URL**: `POST /api/cart/checkout`

**请求字段**
- 无

说明：
- 当前后端会直接读取购物车中 `selected=true` 的商品进行结算预览
- 前端需要先通过 `PUT /api/cart/items/{item_id}` 设置选中状态

**返回字段**
- `200 OK`

| 字段名 | 含义 |
|---|---|
| `cart_id` | 当前购物车 ID |
| `items` | 参与结算的购物车项列表，元素字段见“6.7 购物车项对象” |
| `total_amount` | 结算总金额 |

## 4. 订单

### 4.1 创建订单
- **URL**: `POST /api/orders`

**请求字段**
- 无

说明：
- 当前后端会直接根据购物车中 `selected=true` 的商品创建订单
- 前端需要先在购物车中勾选要下单的商品

**返回字段**
- `201 Created`
- 返回 `Order` 对象，字段见“6.8 订单对象”

### 4.2 获取我的订单列表
- **URL**: `GET /api/orders`

**请求字段**
- 无

**返回字段**
- `200 OK`
- 返回数组，数组元素字段见“6.8 订单对象”

### 4.3 获取订单详情
- **URL**: `GET /api/orders/{order_id}`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `order_id` | 是 | 订单 ID |

**返回字段**
- `200 OK`
- 返回 `Order` 对象，字段见“6.8 订单对象”

### 4.4 支付订单
- **URL**: `POST /api/orders/{order_id}/pay`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `order_id` | 是 | 订单 ID |

**请求字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `payment_method` | 是 | 支付方式 |

**payment_method 可选值**

| 值 | 含义 |
|---|---|
| `wechat` | 微信支付 |
| `alipay` | 支付宝支付 |
| `balance` | 余额支付 |

**返回字段**
- `200 OK`
- 返回支付后的 `Order` 对象，字段见“6.8 订单对象”

### 4.5 取消订单
- **URL**: `POST /api/orders/{order_id}/cancel`

**路径字段**

| 字段名 | 是否必填 | 含义 |
|---|---|---|
| `order_id` | 是 | 订单 ID |

**请求字段**
- 无

**返回字段**
- `200 OK`
- 返回取消后的 `Order` 对象，字段见“6.8 订单对象”

## 5. 常见错误返回

业务校验失败时，当前后端通常返回以下结构：

| 字段名 | 含义 |
|---|---|
| `detail` | 错误描述 |

序列化器校验失败时，后端会按字段名返回错误信息对象。
