# 无人超市管理系统：答辩演示脚本

> 状态：当前实现版
> 作用：用于答辩或阶段汇报时，按固定顺序演示系统功能，保证讲解节奏稳定、流程完整、功能点覆盖充分。

## 1. 演示目标

本次演示重点覆盖以下能力：

- 用户端登录与浏览流程
- 商品浏览、加购、下单、mock 支付
- 用户订单管理
- 商家商品管理
- 图像识别辅助录入
- 库存预警与库存调整
- 公告推荐管理
- 销售统计

## 2. 演示前准备

### 2.1 启动服务

- 后端：`conda run -n shop python .\backend\manage.py runserver`
- 移动端 H5：`cd frontend/mobile && npm run dev`
- Web 端：`cd frontend/web && npm run dev`

### 2.2 准备数据

建议提前准备：

- 1 个普通用户账号
- 1 个商家账号
- 2 到 3 个商品分类
- 3 到 5 个已上架商品
- 1 条已发布公告
- 2 到 3 个推荐商品

### 2.3 演示顺序

推荐顺序：

1. 先演示 Web 用户端主流程
2. 再演示 Web 商家端管理流程
3. 最后补充移动端 H5 页面形态

## 3. 演示脚本

### 3.1 项目背景

讲解要点：

- 本系统是一个本科毕设项目
- 项目聚焦软件系统，不涉及硬件设备
- 系统分为移动端 H5 用户端和 Web 统一管理端
- Web 端根据角色分为普通用户和商家两类权限

可用话术：

“本项目实现的是一个无人超市管理系统，围绕顾客自助购物和商家后台管理两条主线展开。系统不依赖硬件，只聚焦软件业务闭环，包括商品、库存、购物车、订单、公告推荐和图像识别辅助录入等功能。”

### 3.2 用户登录

操作步骤：

- 打开 Web 用户端登录页
- 使用普通用户账号登录
- 登录成功后进入用户首页

讲解要点：

- 系统支持用户名密码登录
- 已实现登录状态校验和路由守卫
- 未登录用户不能直接访问受保护页面

### 3.3 首页浏览

操作步骤：

- 展示首页公告
- 展示推荐商品
- 展示商品分类入口
- 点击推荐商品或进入商品浏览页

讲解要点：

- 首页内容来自后端聚合接口
- 公告默认展示最新一条
- 推荐商品支持后台排序控制首页位置

### 3.4 商品浏览与加购

操作步骤：

- 进入商品浏览页
- 选择分类筛选商品
- 使用关键词搜索商品
- 点击“加入购物车”
- 进入购物车查看商品

讲解要点：

- 商品列表支持分类和关键词筛选
- 商品显示库存状态
- 同一商品重复加入购物车时会自动累加数量

### 3.5 购物车与结算

操作步骤：

- 在购物车中修改商品数量
- 勾选商品进入结算页
- 查看结算金额
- 点击提交订单
- 执行 mock 支付

讲解要点：

- 提交订单时先生成待支付订单
- 支付采用 mock 流程
- 下单时会校验并扣减库存

### 3.6 用户订单管理

操作步骤：

- 进入“我的订单”
- 按状态查看订单
- 演示待支付订单继续支付
- 演示取消订单

讲解要点：

- 用户只能查看自己的订单
- 订单状态包括待支付、已支付、已完成、已取消
- 取消订单后库存会自动回补

### 3.7 切换到商家端

操作步骤：

- 退出当前用户或切换角色
- 使用商家账号登录 Web 管理端
- 展示左侧菜单

讲解要点：

- Web 端采用统一后台式 UI
- 用户和商家共用一套界面壳
- 通过角色权限控制菜单和页面访问范围

### 3.8 商品管理

操作步骤：

- 进入商品管理页
- 查看商品列表
- 新增一个商品
- 编辑商品
- 演示商品上架与下架

讲解要点：

- 商品包含分类、价格、描述、主图、状态等字段
- 下架商品不会在用户端展示

### 3.9 图像识别辅助录入

操作步骤：

- 进入商品新增页
- 上传商品图片
- 调用图像识别接口
- 展示识别结果回填
- 手动补充价格、库存、描述后保存

讲解要点：

- 图像识别采用 YOLO 辅助录入思路
- 当前实现为 mock 接口链路，后续可替换为真实模型
- 系统只做辅助识别，不直接自动上架

### 3.10 库存管理与库存预警

操作步骤：

- 进入库存管理页
- 查看当前库存和预警阈值
- 调整某个商品库存
- 展示低库存商品的预警标记

讲解要点：

- 一个商品对应一条库存记录
- 库存与商品信息分表设计
- 库存低于等于阈值时触发预警
- 库存变更会写入库存日志

### 3.11 公告与推荐位管理

操作步骤：

- 进入公告管理页
- 新增或编辑公告
- 删除公告
- 进入推荐位管理页
- 选择商品加入推荐位
- 修改推荐顺序

讲解要点：

- 首页公告与推荐位由商家后台统一维护
- 推荐位支持排序，影响首页展示顺序

### 3.12 订单管理与销售统计

操作步骤：

- 进入订单管理页
- 查看全部订单并按状态筛选
- 打开某笔已支付订单详情
- 将订单标记为已完成
- 进入销售统计页

讲解要点：

- 商家可以查看全量订单
- 已支付订单可以标记为已完成
- 统计页展示订单总数、销售总额、已完成订单数和热销商品排行

### 3.13 移动端 H5 补充演示

操作步骤：

- 打开移动端 H5 首页
- 展示顶部 + 底部导航布局
- 简单演示商品浏览、购物车和订单页

讲解要点：

- 移动端单独采用购物型 UI
- Web 端采用后台式 UI
- 两者共用同一套后端接口体系

## 5. 常见答辩问题

### 5.1 为什么订单要先生成再支付

回答要点：

- 先生成订单才能有订单号
- 支付失败后订单仍可保留为待支付
- 方便后续做取消订单和库存回补

### 5.2 为什么库存要独立成表

回答要点：

- 商品信息和库存信息职责不同
- 便于做库存预警和库存日志
- 便于下单扣减和取消回补

### 5.3 为什么图像识别只做辅助录入

回答要点：

- 毕设范围需要控制复杂度
- 识别结果需要人工确认更稳妥
- 既体现智能化亮点，又不影响核心业务闭环

### 5.4 为什么 Web 端统一界面

回答要点：

- 降低开发成本
- 保持设计一致性
- 通过角色权限区分用户和商家能力
